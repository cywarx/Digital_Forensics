## 🗂️ The Sleuth Kit (TSK) — The Foundation

> [!tip] What is Sleuth Kit?
> **The Sleuth Kit (TSK)** is a collection of **command-line tools** for investigating disk images. Think of it as a toolbox 🧰 — each tool does one specific job.

### Install Sleuth Kit
```bash
sudo apt install sleuthkit
```

### Key Tools Inside Sleuth Kit

| Tool     | What it Does                    | Simple Analogy                   |
| -------- | ------------------------------- | -------------------------------- |
| `fls`    | Lists files (including deleted) | Directory listing with ghosts 👻 |
| `icat`   | Extracts file content by inode  | Copying a file by its ID number  |
| `istat`  | Shows file metadata             | File's birth certificate 📜      |
| `fsstat` | Shows filesystem info           | Disk's ID card                   |
| `mmls`   | Shows partition layout          | Disk's floor plan 🏢             |

## 📁 fls — File Listing Tool

### What is fls?

`fls` stands for **File Listing**. It lists all files and directories in a disk image — including **deleted ones**!

> [!example] Simple Analogy
> Imagine a library where books are removed from the shelf but their **catalog entry still exists**. `fls` shows you ALL catalog entries — even for removed books.

### Basic Syntax
```bash
fls [options] <image_file>
```

### Common Flags

| Flag | Meaning |
|------|---------|
| `-d` | Show only **deleted** files |
| `-r` | **Recursive** (go into subdirectories) |
| `-l` | Long listing (show more details) |
| `-p` | Show full path |
| `-f` | Specify filesystem type (fat, ntfs, ext4) |

---

### 🖥️ Practical Examples

#### Example 1 — List ALL files
```bash
fls pendrive.img
```
```
# OUTPUT:
r/r 3:    FAT1
v/v 4:    $OrphanFiles
r/r 5:    secret_doc.pdf
r/r * 6:  ab.txt
d/d 7:    photos/
```

> [!note] Reading the Output
> - `r/r` = regular file
> - `d/d` = directory
> - `v/v` = virtual file (system file)
> - `*` ← **THIS STAR means the file is DELETED!**
> - The number (like `6`) is the **inode number** — like a file's ID

---

#### Example 2 — List ONLY Deleted Files
```bash
fls -d pendrive.img
```
```
# OUTPUT:
r/r * 6:    ab.txt
r/r * 9:    passwords.txt
r/r * 11:   photo.jpg
```

---

#### Example 3 — Recursive Listing (includes subfolders)
```bash
fls -r pendrive.img
```
```
# OUTPUT:
r/r 5:      secret_doc.pdf
d/d 7:      photos/
r/r 8:      photos/vacation.jpg
r/r * 6:    ab.txt
r/r * 10:   photos/deleted_pic.png
```

---

#### Example 4 — Long Listing with Details
```bash
fls -l pendrive.img
```
```
# OUTPUT:
r/r 5:    secret_doc.pdf    1024    Wed Apr  1 06:30:00 2026
r/r * 6:  ab.txt            17      Wed Apr  1 06:25:00 2026
d/d 7:    photos/           0       Tue Mar 31 20:00:00 2026
```

---

#### Example 5 — Specify Filesystem Type
```bash
fls -f fat32 pendrive.img
fls -f ntfs windows_drive.img
fls -f ext4 linux_drive.img
```

## 📤 icat — File Extractor by Inode

### What is icat?

`icat` stands for **Inode Concatenate**. It reads a file from a disk image using its **inode number** and outputs the content.

> [!example] Simple Analogy
> Think of inode numbers like **hotel room numbers** 🏨. Even if the guest (file) has checked out (deleted), if the room (data block) hasn't been cleaned (overwritten), you can still walk in and see everything.

### Basic Syntax
```bash
icat <image_file> <inode_number> > output_file
```

---

### 🖥️ Practical Examples

#### Example 1 — Recover a Deleted Text File
```bash
# Step 1: Find deleted files
fls -d pendrive.img
```
```
# OUTPUT:
r/r * 6:    ab.txt
```

```bash
# Step 2: Extract the file using its inode number (6)
icat pendrive.img 6 > recovered_ab.txt

# Step 3: Read it
cat recovered_ab.txt
```
```
# OUTPUT:
this is a secret
```

---

#### Example 2 — Recover a Deleted Image
```bash
fls -d -r photo_drive.img
```
```
# OUTPUT:
r/r * 14:   selfie.jpg
r/r * 15:   screenshot.png
```

```bash
# Recover the JPEG
icat photo_drive.img 14 > recovered_selfie.jpg

# Verify it's a valid image
file recovered_selfie.jpg
```
```
# OUTPUT:
recovered_selfie.jpg: JPEG image data, JFIF standard 1.01
```

---

#### Example 3 — Recover and Check File Hash
```bash
icat pendrive.img 6 > recovered.txt

# Generate MD5 hash to verify integrity
md5sum recovered.txt
```
```
# OUTPUT:
d8e8fca2dc0f896fd7cb4cb0031ba249  recovered.txt
```

---

#### Example 4 — Recover to Specific Directory
```bash
mkdir recovered_files
icat pendrive.img 6 > recovered_files/ab.txt
icat pendrive.img 14 > recovered_files/selfie.jpg
ls recovered_files/
```
```
# OUTPUT:
ab.txt  selfie.jpg
```

> [!warning] When icat Returns Empty
> If `icat` returns nothing, the **data blocks were overwritten**. Try `foremost` or `scalpel` for carving instead.

## 🔬 istat — Inode Inspector

### What is istat?

`istat` shows you the **metadata** of a file — timestamps, size, which blocks it uses — without extracting content.

> [!example] Simple Analogy
> Like reading a file's **birth certificate** before deciding to recover it.

### 🖥️ Practical Examples

```bash
istat pendrive.img 6
```
```
# OUTPUT:
inode: 6
Allocated
Group: 0
Generation Id: 0
uid / gid: 0 / 0
mode: rrwxrwxrwx
Flags: 0x0
size: 17
num of links: 0

Inode Times:
Accessed:       2026-04-01 06:25:10 (UTC)
File Modified:  2026-04-01 06:24:55 (UTC)
Inode Modified: 2026-04-01 06:40:10 (UTC)
Created:        2026-04-01 06:20:00 (UTC)

Direct Blocks:
2050
```

> [!note] Key Fields to Check
> - **size: 17** → File has 17 bytes of data (recoverable!)
> - **size: 0** → Data is gone (not recoverable with icat)
> - **Direct Blocks** → The actual disk block where data lives
> - **num of links: 0** → File is deleted (no directory points to it)

## 🕵️ foremost — File Carver

### What is foremost?

`foremost` is a **file carver** — it scans through raw disk data looking for **file signatures (magic bytes)** and reconstructs files.

> [!example] Simple Analogy
> Imagine shredded documents 📄. `foremost` is like someone who recognizes "this piece starts with a PDF header" and reassembles it — even without a file table entry.

### How File Carving Works
```
Disk Data (raw bytes):
[...garbage...][FFD8FFE0...][...jpeg data...][FFD9][...garbage...][25504446...][...pdf data...]
                 ↑                                                  ↑
           JPEG header found!                               PDF header found!
           Extract until footer                             Extract until footer
```

### Common File Signatures foremost knows

| File Type | Header (Magic Bytes) | Footer |
|-----------|---------------------|--------|
| JPEG | `FF D8 FF` | `FF D9` |
| PDF | `25 50 44 46` (`%PDF`) | `%%EOF` |
| ZIP | `50 4B 03 04` (`PK..`) | — |
| PNG | `89 50 4E 47` | `AE 42 60 82` |
| MP3 | `49 44 33` (`ID3`) | — |
| DOCX | `50 4B 03 04` (ZIP) | — |

---

### 🖥️ Practical Examples

#### Example 1 — Basic File Carving
```bash
foremost -i pendrive.img -o recovered/
```
```
# OUTPUT:
Processing: pendrive.img
|*|
```

```bash
ls recovered/
```
```
# OUTPUT:
audit.txt  jpg/  pdf/  png/  zip/
```

```bash
cat recovered/audit.txt
```
```
# OUTPUT:

Audit File
Foremost started at Wed Apr  1 06:40:25 2026

Configuration file: /etc/foremost.conf

File: pendrive.img
Start: Wed Apr  1 06:40:25 2026
Length: 25 MB (26214400 bytes)

Num    Name (bs=512)     Size    File Offset    Comment
0:     00000100.jpg      2048    51200          (1920 x 1080)
1:     00000200.pdf      4096    102400

Finish: Wed Apr  1 06:40:26 2026
2 FILES EXTRACTED
```

---

#### Example 2 — Carve Only Specific File Types
```bash
# Only look for JPEGs and PDFs
foremost -t jpg,pdf -i pendrive.img -o recovered/
```
```
# OUTPUT:
Processing: pendrive.img
|*|
```

---

#### Example 3 — Verbose Mode (see what's happening)
```bash
foremost -v -i pendrive.img -o recovered/
```
```
# OUTPUT:
Processing: pendrive.img
Searching with 7 headers
JPEG Header  found at offset 51200
JPEG Footer  found at offset 102400 (Size: 51200 bytes)
PDF  Header  found at offset 204800
|*|
```

---

#### Example 4 — Carve from a Specific Offset
```bash
foremost -s 1024 -i pendrive.img -o recovered/
```

---

#### Example 5 — Check Recovered Files
```bash
ls -la recovered/jpg/
```
```
# OUTPUT:
total 64
-rw-r--r-- 1 root root  2048 Apr  1 06:40 00000100.jpg
-rw-r--r-- 1 root root  4096 Apr  1 06:40 00000200.jpg

file recovered/jpg/00000100.jpg
```
```
# OUTPUT:
recovered/jpg/00000100.jpg: JPEG image data, JFIF standard 1.01, resolution (DPI), density 96x96
```

> [!warning] Why foremost finds 0 files for .txt
> Plain text files have **no magic bytes**. foremost can't find them!
> Use `icat` + `fls` for text files instead.

## 🔩 binwalk — Firmware & Binary Analyzer

### What is binwalk?

`binwalk` is a tool for analyzing **binary files and firmware images**. It finds embedded files, compressed data, and hidden content inside binary blobs.

> [!example] Simple Analogy
> Like an **X-ray machine** 🩻 for files. You point it at any file and it tells you everything hidden inside — compressed archives, images, code, certificates.

### Install binwalk
```bash
sudo apt install binwalk
# or
pip install binwalk
```

---

### 🖥️ Practical Examples

#### Example 1 — Basic Scan (What's inside?)
```bash
binwalk firmware.bin
```
```
# OUTPUT:
DECIMAL       HEXADECIMAL     DESCRIPTION
------------------------------------------------------------
0             0x0             Linux kernel ARM boot executable zImage
1048576       0x100000        Squashfs filesystem, little endian, 4.0
2097152       0x200000        JFFS2 filesystem, little endian
3145728       0x300000        gzip compressed data, max compression
```

---

#### Example 2 — Scan a Disk Image
```bash
binwalk pendrive.img
```
```
# OUTPUT:
DECIMAL       HEXADECIMAL     DESCRIPTION
------------------------------------------------------------
0             0x0             DOS/MBR boot sector
512           0x200           FAT 16 BIOS Parameter Block
5632          0x1600          JPEG image data, JFIF standard 1.01
57344         0xE000          Zip archive data, "secret.txt"
```

---

#### Example 3 — Extract Everything Automatically
```bash
binwalk -e firmware.bin
```
```
# OUTPUT:
DECIMAL       HEXADECIMAL     DESCRIPTION
------------------------------------------------------------
0             0x0             Linux kernel ARM boot executable zImage
1048576       0x100000        Squashfs filesystem, little endian, 4.0

Extracted files saved to: _firmware.bin.extracted/
```

```bash
ls _firmware.bin.extracted/
```
```
# OUTPUT:
100000.squashfs  squashfs-root/  300000.gz  300000
```

---

#### Example 4 — Recursive Extraction (go deeper)
```bash
binwalk -eM firmware.bin
# -M = matryoshka mode (extract → scan extracted → extract again)
```

---

#### Example 5 — Entropy Analysis (detect encryption/compression)
```bash
binwalk -E firmware.bin
```
```
# OUTPUT:
DECIMAL       HEXADECIMAL     ENTROPY
------------------------------------------------------------
0             0x0             Rising entropy edge (0.857212)
1048576       0x100000        Falling entropy edge (0.121345)
```

> [!tip] Reading Entropy
> - **High entropy (~0.8–1.0)** = encrypted or compressed data
> - **Low entropy (~0.0–0.3)** = plain text or empty space
> - **Rising edge** = start of compressed/encrypted section
> - **Falling edge** = end of compressed/encrypted section

---

#### Example 6 — Extract a Specific File by Offset
```bash
# Extract 1024 bytes starting at offset 57344
dd if=pendrive.img bs=1 skip=57344 count=1024 of=extracted_zip.zip
file extracted_zip.zip
```
```
# OUTPUT:
extracted_zip.zip: Zip archive data
```

---

#### Example 7 — Signature Scan Only (no extraction)
```bash
binwalk -B pendrive.img
```

## 🖥️ Autopsy — GUI Forensics Platform

### What is Autopsy?

**Autopsy** is a **graphical user interface (GUI)** built on top of The Sleuth Kit. It's like having all forensic tools in one beautiful dashboard.

> [!example] Simple Analogy
> If Sleuth Kit tools (`fls`, `icat`) are individual **hand tools** 🔧, Autopsy is the full **workshop with everything organized** 🏭.

### Install Autopsy
```bash
sudo apt install autopsy
# Launch in browser
autopsy
# Opens at: http://localhost:9999/autopsy
```

### OR use standalone Autopsy (Windows/Linux)
```
Download from: https://www.autopsy.com/download/
```

## ⚡ Quick Reference Cheat Sheet

```bash
# ═══════════════════════════════════════════
#           DIGITAL FORENSICS CHEATSHEET
# ═══════════════════════════════════════════

# 📸 CREATE IMAGE
sudo dd if=/dev/sdb of=drive.img bs=512 status=progress

# 📋 LIST ALL FILES
fls drive.img

# 👻 LIST DELETED FILES ONLY
fls -d drive.img

# 🔁 RECURSIVE LIST
fls -r -l drive.img

# 📄 FILE METADATA
istat drive.img <inode>

# 💾 RECOVER FILE
icat drive.img <inode> > output_file

# 🔍 CARVE ALL FILE TYPES
foremost -i drive.img -o output_dir/

# 🎯 CARVE SPECIFIC TYPES
foremost -t jpg,pdf,zip -i drive.img -o output_dir/

# 🧬 ANALYZE BINARY
binwalk drive.img

# 📦 EXTRACT FROM BINARY
binwalk -e drive.img

# 🌊 ENTROPY ANALYSIS
binwalk -E drive.img

# 🖥️  LAUNCH AUTOPSY GUI
autopsy

# 🔐 HASH IMAGE
md5sum drive.img
sha256sum drive.img
sha1sum drive.img

# 📊 FILESYSTEM INFO
fsstat drive.img

# 🗺️  PARTITION TABLE
mmls drive.img
```

## 🧪 Practice Lab Setup

### Create Your Own Test Image
```bash
# Step 1: Create a blank 25MB image
dd if=/dev/zero of=test.img bs=1M count=25
```
```
# OUTPUT:
25+0 records in
25+0 records out
26214400 bytes (26 MB, 25 MiB) copied, 0.045 s, 583 MB/s
```

```bash
# Step 2: Format it as FAT16
mkfs.fat -F 16 test.img
```
```
# OUTPUT:
mkfs.fat 4.2 (2021-01-31)
```

```bash
# Step 3: Mount and add test files
mkdir /tmp/testmount
sudo mount -o loop test.img /tmp/testmount

echo "secret password: hunter2" | sudo tee /tmp/testmount/passwords.txt
echo "this is a secret" | sudo tee /tmp/testmount/ab.txt
sudo cp /usr/share/pixmaps/*.png /tmp/testmount/ 2>/dev/null || true

sudo umount /tmp/testmount
```

```bash
# Step 4: Delete the files (simulate deletion)
sudo mount -o loop test.img /tmp/testmount
```
