```
```
# OUTPUT:
r/r 3:        HEXX_USB    (Volume Label Entry)
r/r 4:        readme.txt
r/r 7:        employee_badge.jpg
r/r 8:        system.log
r/r 11:       project_files.zip

v/v 1046468:  $FAT1
v/v 1046469:  $FAT2

```

> [!warning] FAT Name Mangling
> FAT16 marks deleted files by overwriting the first byte of the filename with `0xE5`.
> That is why `flag.ctf` appears as `_lag.ctf`. **The actual data is untouched.**

> [!tip] Inode numbers may differ slightly between runs depending on cluster alignment. Always check with `fls` first.

### 1d — List DELETED files only

```
```
# OUTPUT:
r/r * 14:     secret_notes.txt
r/r * 17:     secret_plan.png
r/r * 20:     _lag.ctf
```

**All 3 deleted files found!**

## 🖼️ Phase 4 — Image Carving with foremost

> [!tip] Carving bypasses the filesystem entirely — scans raw bytes for magic signatures. Works even when inodes are wiped.

```bash
foremost -v -t jpg,png,zip -i pendrive.img -o carved/
```
```

JPEG Header found at offset ...
ZIP  Header found at offset ...
PNG  Header found at offset ...
```

```bash
ls carved/jpg/ carved/png/ carved/zip/
```

```bash
file carved/jpg/* carved/png/* carved/zip/*
```
```
# OUTPUT:
carved/jpg/00000xxx.jpg: JPEG image data, 400x300   <- employee_badge
carved/png/00000xxx.png: PNG image data, 300x200    <- secret_plan
carved/zip/00000xxx.zip: Zip archive data           <- project_files
```

> [!warning] Why foremost misses .ctf files
> Plain text files have no magic bytes — foremost cannot detect them.
> **Always combine foremost with icat for complete recovery.**

### Extract the ZIP
```bash

cat carved/zip_contents/data.txt
```

## 🔩 Phase 5 — Deep Binary Scan with binwalk

### Full signature scan
```bash
binwalk pendrive.img
```
```
# OUTPUT:
DECIMAL       HEXADECIMAL     DESCRIPTION
--------------------------------------------------------------
0             0x0             DOS/MBR boot sector
1048576       0x100000        FAT 16 BIOS Parameter Block
...           ...             JPEG image data  (employee_badge)
...           ...             Zip archive data
...           ...             PNG image data   (secret_plan)
```

### Entropy analysis
```bash
binwalk -E pendrive.img
```

> [!tip] Entropy Reading Guide
> - Near 0.0 = empty or plain zeros
> - Near 0.5 = plain text
> - Above 0.8 = compressed / encrypted data
> - Rising edge = start of something interesting

### Extract everything
```bash
binwalk -e pendrive.img
ls _pendrive.img.extracted/
```

## ⚠️ Phase 6 — The Extension Trick

> One file on the drive is NOT what its extension claims.

### Find the fake PDF
```bash
icat pendrive.img 12 > decoy.pdf
file decoy.pdf
```

> [!info] No `decoy.pdf` in v3.0 script
> The current `pendrive.py` does **not** include `decoy.pdf` in the visible file list.
> To add this challenge, insert the following into `make_files()` and `visible` list in `build_image()`:

```python
# In make_files() — add a JPEG disguised as PDF:
img3 = Image.new("RGB", (200, 150), (200, 200, 200))
d3 = ImageDraw.Draw(img3)
d3.text((30, 60), "I am NOT a PDF!", fill=(255, 0, 0))
img3.save(out / "decoy.pdf", "JPEG")   # saved as JPEG but with .pdf name
```

```python
# In build_image() — add to visible list:
visible = ["readme.txt", "employee_badge.jpg", "system.log", "project_files.zip", "decoy.pdf"]
```

### After adding it — detecting the trick:
```bash
icat pendrive.img <inode> > decoy.pdf
file decoy.pdf
```
```
# OUTPUT:
decoy.pdf: JPEG image data, JFIF standard 1.01, 200x150, components 3
```

```bash
xxd decoy.pdf | head -2
```
```
# JPEG magic bytes (NOT a PDF):
00000000: ffd8 ffe0 0010 4a46 4946 ...
# A real PDF would start with: 25504446 (%PDF)
```

### Magic bytes reference
```
JPEG  →  FF D8 FF
PNG   →  89 50 4E 47  (.PNG)
PDF   →  25 50 44 46  (%PDF)
ZIP   →  50 4B 03 04  (PK..)
```

> [!tip] Always run `file` on suspicious files. **Never trust the extension alone!**
